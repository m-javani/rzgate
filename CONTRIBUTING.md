## What We Accept

We welcome:
- Bug fixes
- Performance improvements
- Documentation updates
- Test coverage improvements

We may not accept:
- Major architectural changes (please open an issue first)
- Breaking changes without prior discussion
- Contributions that conflict with commercial interests

## Process

1. Open an issue to discuss your proposed change
2. Fork the repository
3. Create a feature branch (`git checkout -b feature/amazing-fix`)
4. Commit your changes (`git commit -m 'Fix: description'`)
5. Push to your fork (`git push origin feature/amazing-fix`)
6. Open a Pull Request

## Code Style

- Follow standard Go conventions (`gofmt`)
- Write tests for new functionality
- Update documentation as needed
